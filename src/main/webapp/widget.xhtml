Widget Tag

Tiefaces support multiple Primeface input widget for cell input method (i.e. calandar, dropdown, inputNumber, inputArea..).
Use widget command to define the widget name and attributes:

How to define?

Use the special syntax below defined in the comment for the staring cell of the command.

...
$widget.calendar{showOn="button" pattern="yyyy/MM/dd" readonlyInput="true"}
$widget.dropdown{itemLabels="Male;Female" itemValues="M;F" }
$widget.inputnumber{symbol=" years" symbolPosition="s" minValue="0" maxValue="999" decimalPlaces="2"}
...

Note:
1. One cell only can contain one widget tag.
2. Each command in the comment must remain in the same line, don't use enter key to divide it.

The Attributes of $widget.calendar

Can use any attribute from p:calendar. See Primeface p:calendar for detail.

The Attributes of $widget.inputnumber

Can use any attribute from p:inputNumber. See Primeface p:calendar for detail.

The Attributes of $widget.dropdown

  • itemLabels: list of item's labels. use ";" to seperate.
  • itemValues: list of item's values. use ";" to seperate.
  • defaultItemLabel: default item's labels.
  • defaultItemValue: default item's value.

Can also use any attribute applied to p:selectOneMenu (not for f:selectItem). See Primeface p:calendar for detail.
